## 第一部分：回测系统全面测试计划

### 1.1 当前状态诊断

**已有覆盖（强）**：
- 回测引擎核心 (engine.test.ts, 1,068 行)
- 统计指标 (statistics.test.ts, 1,010 行)
- 信号扫描 (signal-scanner.test.ts, 1,119 行)
- 输入校验 (validators.test.ts, 610 行)
- 金融数学 (financial-math.test.ts, 594 行)
- 评分计算 (score-calculator.test.ts, 658 行)

**关键缺口（须补）**：

| 缺口 | 风险等级 | 影响范围 |
|------|---------|---------|
| 数据库集成测试（db-kline-provider.ts） | **高** | 实盘数据获取失败用户感知最直接 |
| 三层配额中间件（quota-check.ts, 306 行）无测试 | **高** | 付费系统根基，错扣/漏扣直接影响收入和信任 |
| API 路由端到端（backtest/route.ts 等 6 条路由） | **高** | 前后端数据格式不匹配是最常见线上问题 |
| 数据持久化（kline-persister.ts）异步无测试 | **中** | 静默失败导致同一股票反复调 API |
| Identity 服务超时降级路径 | **中** | 1.5s 超时 fail-open 时配额形同虚设 |
| 大规模投资组合压测（50+ 股票） | **中** | Worker Pool 启动开销未度量 |
| Mock 数据偏差（固定 volatility=0.02） | **低** | 模拟模式下策略表现失真 |

### 1.2 测试分层架构

```
L4  E2E / 冒烟测试          ← Playwright 4 viewport（已有框架）
L3  API 集成测试             ← 路由 + 数据源 + 配额联动（缺失）
L2  服务集成测试             ← DB + Redis + Identity 交互（缺失）
L1  单元测试                 ← 引擎/统计/校验/数学（已覆盖 ~85%）
```

## 第二部分：用户付费方案

### 2.1 定价哲学

**核心原则**：让用户"垫垫脚就能够到"

- 免费层让用户完整体验核心价值（回测一只股票，看到完整报告）
- 付费门槛设在"想多做一点"的时刻（多股验证、AI 顾问、更多历史数据）
- 每一层付费都给用户清晰可感知的"多出来的东西"
- 避免"面议"——透明定价建立信任

### 2.2 竞品定价参考

| 平台 | 免费 | 入门付费 | 高级付费 | 模式 |
|------|------|---------|---------|------|
| 果仁网 | 基础回测 | 90 元/月 (VIP) | 583 元/月 (至尊) | 订阅 |
| BigQuant | 基础回测+AI | 129 元/月 | 659 元/月 | 订阅 |
| 同花顺 SuperMind | 秒级回测 | ~100-170 元/月 | 50 万资金门槛实盘 | 混合 |
| 聚宽 | 回测+教育 | 数百元/月 (数据) | B 端面议 | 数据+订阅 |
| 掘金 | 基本投研 | 817 元/月 (Pro) | 券商版 | 订阅 |

**Gushen 定位**：对标果仁 VIP / BigQuant Standard，低于掘金 Pro。差异化在于 "AI 原生 + 无需编程 + 更低起步价"。

### 2.5 付费转化钩子设计

**原则**：用户在"心流"中遇到限制，而非冷启动时。

| 触发时机 | 钩子 | 转化层级 |
|---------|------|---------|
| 回测完成，想多验证几只 | "一键验证沪深 300 成分股" 按钮 → 升级提示 | Explorer → Trader |
| 看完结果，想问 AI 这策略怎么样 | "AI 帮你诊断策略弱点" 入口 → 升级提示 | Explorer → Trader |
| Trader 想看辩论模式 | "多空辩论" 按钮锁定 → Pro 预览 | Trader → Pro |
| Trader 想导出 PDF 给客户 | "生成专业报告" → Pro 预览 | Trader → Pro |

| 浏览策略市场想订阅 | "订阅此策略" → 升级提示 | Explorer → Trader |

## 第三部分：UX 重设计方案

### 3.1 设计原则

#### 原则一：信息发现粒度——渐进式披露

**问题**：当前页面一次性展示 30+ 指标，新用户认知过载，专业用户找不到关键数据。

**方案**：三层信息架构

```
第一层（5 秒看懂）：3 个核心数字 + 收益曲线
第二层（30 秒深入）：关键指标卡片组 + 交易统计
第三层（专业钻取）：完整指标表 + 对比分析 + 诊断报告
```

对标：米筐的回测结果页被公认"视觉设计最佳"——核心在于**层次清晰**，不是指标少。

#### 原则二：用户先验知识——垫垫脚够得到

**认知分级**：

| 用户类型 | 先验知识 | 共鸣点（不解释） | 垫脚点（简短解释即可） |
|---------|---------|----------------|---------------------|
| 股民小白 | K 线、涨跌、PE | 涨了多少、亏了多少 | 夏普比率（"每承担 1 分风险赚了几分钱"） |
| 进阶投资者 | 技术指标、回撤 | 最大回撤、胜率 | 索提诺比率（"只看亏损风险的夏普"） |
| 专业量化 | 阿尔法/贝塔、VaR | 所有指标 | 策略诊断的具体建议 |

**实施**：每个指标旁有 `?` 图标，hover 显示 1 行解释 + "了解更多"链接。解释文案不超过 15 个字。

#### 原则三：多步骤舒适感——给用户"掌控感"

**参考**：果仁网的向导式流程（但我们用 AI 替代手动配置）。

```
当前：策略输入 → 生成代码 → 回测 → 结果（扁平，用户不知道"然后呢"）

重设计：
  步骤 1：策略构思   → "告诉我你的想法"
  步骤 2：策略生成   → "AI 帮你写好了"
  步骤 3：参数调优   → "微调到满意"
  步骤 4：单股回测   → "先看一只"
  步骤 5：多股验证   → "放到更多股票上试试"
  步骤 6：AI 诊断    → "看看哪里还能改进"
  步骤 7：发布/保存  → "保存到我的策略库"
```

每步完成后，顶部 stepper 亮起，右侧出现"下一步建议"卡片。用户随时可跳步，但始终知道"我在哪，还可以去哪"。

#### 原则四：专业感来自克制，不是堆砌

**反面教材**：把 30 个指标全放一屏 ≠ 专业。聚宽的回测结果页指标密度高但排布杂乱。

**正面参考**：
- 米筐：指标分组（收益、风险、交易），每组 4-5 个，留白充足
- 同花顺 SuperMind：核心数字大字体，辅助数字小字体
- TradingView：暗色主题 + 金融数据 mono 字体 + 语义色（红涨绿跌）

### 3.3 新手引导流程（Onboarding）

**参考**：Robinhood/Moomoo 的引导式体验（非"自己看文档"模式）

#### 第一次访问

```
步骤 1/3：你的投资经验？
  ○ 刚开始了解量化投资（推荐：从模板开始）
  ○ 有一些股票交易经验（推荐：自然语言策略）
  ○ 熟悉量化交易和编程（推荐：直接编写代码）

步骤 2/3：你最关心什么？
  ○ 稳定收益，控制回撤
  ○ 高收益，能承受波动
  ○ 学习量化，探索策略

步骤 3/3：30 秒体验
  → 根据前两步自动选择一个模板
  → 自动填充参数 + 自动运行回测
  → 3 秒后展示结果（有数据动画）
  → "这是你的第一个量化策略！接下来你可以..."
```

**设计意图**：
- 3 步 30 秒内看到回测结果——**最短时间到价值**
- 投资经验选择决定后续界面的默认信息层级（小白看简化版，专业看完整版）
- 关心点选择影响推荐的模板和顾问风格
- 第一个回测结果是自动运行的，用户零操作成本

### 3.5 移动端适配策略

**竞品空白**：8 家竞品中仅同花顺 i问财有移动 App，这是蓝海。

**短期（响应式）**：
- 策略工坊在移动端采用全屏单步模式（一步占满一屏，滑动切换）
- 回测结果移动端只显示英雄区 + 收益曲线，指标卡片折叠
- 策略市场卡片单列布局

**中期（PWA）**：
- 添加 PWA manifest，支持"添加到主屏幕"
- 推送通知：策略回测完成、市场订阅策略调仓信号

**长期（小程序/App）**：
- 微信小程序（触达渠道最广）
- React Native App（完整体验）

## 附录：关键决策摘要

| 决策 | 选择 | 理由 |
|------|------|------|
| 免费层是否限制指标数量 | 不限制 | 让用户感受专业价值，限制的是"使用深度"（多股、AI）而非"信息深度" |
| 定价起步 | 49 元/月 | 低于果仁 VIP (90元)，接近一杯咖啡/周，心理门槛低 |
| 策略市场分润 | 作者 70% | 高于行业（AppStore 70%），吸引优质策略作者入驻 |
| 信息分层 | 3 层渐进 | 避免竞品通病——新用户认知过载或专业用户信息不足 |
| 步骤设计 | 7 步引导 | 比果仁（7 步配置式）更自然——每步有 AI 辅助，不是手动填表 |
| 移动端 | 先响应式再 PWA | 竞品几乎全是 PC-only，响应式就已经是差异化 |
| 年付折扣 | 约 20% | 49→39, 149→119，鼓励长期但不过度激进 |
